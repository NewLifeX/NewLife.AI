# NewLife.AI — 需求规格说明

> 版本：v3.0 | 日期：2026-04-17

---

## 1. 背景与目标

### 1.1 系统定位

NewLife.AI 是一套**面向 .NET 生态的开源 AI 应用平台**：既提供可嵌入任意 .NET 项目的基础 SDK 库，也提供开箱即用的完整 Web 对话应用。

| 定位 | 说明 |
|------|------|
| **AI 基础库** | `IChatClient` 统一接口、46 个主流服务商适配、`ChatClientBuilder` 中间件管道、Planner、MultiAgent、MCP 客户端/服务端 |
| **对话前端** | 完整的多模型对话 Web 界面，支持流式输出、多模态、函数调用、工具调用可视化 |
| **统一 AI 网关** | 对外提供兼容 OpenAI / Anthropic / Gemini 三大标准协议的 API，便于第三方系统接入 |
| **Agent 平台** | 原生 .NET 工具注册、MCP 协议、Planner 规划器、多智能体协作，快速构建 Agent 化应用 |

### 1.2 工程分层

代码采用三层工程结构：

| 工程 | 定位 | 目标框架 | NuGet |
|------|------|---------|-------|
| `NewLife.AI` | 核心 SDK（46 客户端 / Tools / MCP / Agents / Planner / Filters / Channels / Memory / Pipeline） | netstandard2.1 | `NewLife.AI` |
| `NewLife.ChatData` | 数据层（核心实体 + 基础服务：ChatPipeline / MessageFlow / SkillService / MemoryService / UsageService / ModelService / McpClientService） | netstandard2.1 | `NewLife.ChatData` |
| `NewLife.ChatAI` | ASP.NET Core Web 应用（20 个 API 控制器 + 网关 + 魔方管理后台 + 内嵌 React 前端） | net8.0 / net10.0 | `NewLife.ChatAI` |
| `NewLife.AI.Extensions` | ASP.NET Core DI + MCP over HTTP 扩展 | net6.0 / net8.0 / net10.0 | `NewLife.AI.Extensions` |

### 1.3 核心目标（G1~G8）

本需求文档采用 **G1~G8** 编号体系系统化描述各功能目标。NewLife.AI 聚焦基础与通用能力，**G6 桌面客户端暂不包含**、**G4 知识进化仅提供基础版**。

| 编号 | 目标 | NAI 开源版支持状态 |
|------|------|-------------------|
| **G1** | AI 客户端库（46 服务商 / 5 协议 / 函数调用 / MCP / 多模态） | ✅ 完整 |
| **G2** | WebUI 对话系统（多轮对话 / 流式 / 思考模式 / 多模态 / 技能 / 分享） | ✅ 完整（对标主流 AI 对话平台的基础能力）|
| **G3** | 能力扩展（技能系统 / 工具 / MCP / 过滤器链 / 规划器 / 多智能体） | ✅ 完整 |
| **G4** | 知识进化（用户记忆 10 类 / 对话分析 / 推荐问题） | 🔧 **基础版**（知识蒸馏、知识图谱、知识库 RAG 等高级能力暂不包含）|
| **G5** | API 网关（OpenAI / Anthropic / Gemini / Responses / Images 多协议 + AppKey + 429 重试） | ✅ 完整 |
| **G6** | 桌面客户端（本地工具执行 + 工具审批） | ⬜ **暂不包含**（可通过 `IToolApprovalProvider` 接口自行实现）|
| **G7** | 办公平台集成（钉钉 / 企微 / 飞书 / Webhook） | ✅ 接口 + 4 渠道实现（`NewLife.AI.Channels`）|
| **G8** | 智能体框架（ConversableAgent / GroupChat / ParallelGroupChat / DelegatingAgent / AgentAsTool + Planner） | ✅ 完整 |

### 1.4 版本约束与扩展点

以下能力在当前版本中**暂不包含**，但架构已为其预留了完整扩展点：

- **桌面宿主客户端**：提供 `IToolApprovalProvider` 接口钩子，可在工具执行前实现弹窗审批 UI
- **L4 知识进化高级服务**：`MemoryService` 可继承扩展知识蒸馏（Dream）、痛觉记忆（Nociception）、好奇心探索（Curiosity）、知识图谱等能力，接口均已预留
- **知识库 RAG**：`IChatFilter` 接口可挂入知识检索注入，`IVectorStore` 抽象已提供
- **管道增强**：`ChatPipeline` 可继承叠加轻量路由、OpenTelemetry 埋点、高级上下文压缩等能力
- **IContextEnricher 内置实现**：接口已定义，可自行实现上下文截断、压缩、注入策略（如 EarlyTruncation / Sandwich / Compaction 模式）
- **渠道管理后台页**、**多智能体管理页**、**定时对话任务（ChatJob）**

## 2. 用户角色

| 角色 | 说明 | 核心诉求 |
|------|------|---------|
| **普通用户** | 登录后使用 Web 对话 | 流畅对话、模型切换、会话持久化 |
| **访客** | 通过分享链接查看只读会话 | 分享链接访问、只读浏览 |
| **API 开发者** | 通过 OpenAI 标准协议接入 | 兼容 API、AppKey 认证、用量统计 |
| **.NET 开发者** | 在自有项目中嵌入 NewLife.AI SDK | NuGet 引用、`IChatClient` 简洁 API、支持 DI |
| **管理员** | 通过魔方后台管理模型/AppKey/用户 | 系统配置、使用量分析 |

> 本系统不负责登录鉴权，由 `NewLife.Cube` 等外部权限框架统一处理，后端通过当前会话获取 `UserId`。

## 3. 功能需求

### 3.1 G1 — AI 客户端库

NewLife.AI 作为系统基础设施层，为上层所有功能提供统一的模型调用能力。

**功能要点**：

- **46 个主流 AI 服务商**：通过反射扫描 `[AiClient]` 特性自动注册到 `AiClientRegistry`，零配置新增；按来源分为：
  - **9 个专属协议客户端**：OpenAI、DeepSeek、NewLifeAI、AzureAI、Anthropic、Gemini、DashScope、Ollama、Bedrock
  - **37 个 OpenAI 兼容厂商**：在 `BuiltinChatClient.cs` 集中聚合（字节豆包 / 智谱 / Moonshot / 混元 / 千帆 / 星火 / Mistral / Groq / Perplexity / OpenRouter 等）
- **5 种原生协议 + 1 种签名方案**：
  - OpenAI ChatCompletions（`OpenAIChatClient`，37 兼容厂商复用）
  - Anthropic Messages（`AnthropicChatClient`）
  - Google Gemini（`GeminiChatClient`）
  - 阿里 DashScope（`DashScopeChatClient`）
  - Ollama（`OllamaChatClient`）
  - AWS Bedrock Converse API + **SigV4 签名**（`BedrockChatClient`）
- **统一 `IChatClient` 接口**：对齐 Microsoft MEAI 规范，`GetResponseAsync()` + `GetStreamingResponseAsync()` 双模式
- **`ChatClientBuilder` 中间件管道**：链式 `UseFilters()` / `UseTools()` / `Build()`
- **原生函数调用**：`[ToolDescription]` 反射 + `ToolSchemaBuilder` 自动 JSON Schema + `ToolChatClient` 多轮工具循环
- **MCP 协议双向支持**：客户端对接外部 MCP 服务器（stdio / HTTP SSE 传输），服务端通过 `NewLife.AI.Extensions.AspNetMcpServer` 将本系统工具暴露
- **思考模式（Thinking Mode）**：Auto / Think / Fast 三档，支持交错思考（think–tool–think）
- **多模态**：图像理解（Vision）、图像生成（Text-to-Image）、图像编辑（Inpainting）
- **Planner 规划器**：`FunctionCallingPlanner` 将自然语言目标分解为多步工具计划
- **MultiAgent 框架**：`ConversableAgent`、`GroupChat`（轮询 / LLM 选择器）、`ParallelGroupChat`、`DelegatingAgent`、`AgentAsTool`
- **语义记忆抽象**：`ISemanticMemory` + `IVectorStore` + 内存实现开箱即用
- **向量嵌入**：`IEmbeddingClient` 接口
- **特别适配阿里百炼**：`enable_search` / 图像生成等扩展参数透传
- **兼容性**：netstandard2.1，实际可在 .NET 4.5 至 .NET 10 上运行

### 3.2 G2 — WebUI 对话系统

Web 对话界面（React 19 + TypeScript + Vite + Zustand）对标主流 AI 对话平台的通用能力。

**功能要点**：

- **对话基础**：多轮对话、流式输出、停止生成、重新生成、编辑消息继续对话
- **会话管理**：会话列表（分组 / 搜索 / 分页加载）、新建 / 删除 / 重命名 / 置顶、首条消息后自动生成标题
- **模型切换**：运行时选择不同模型，支持 46 家服务商的多种模型
- **思考模式**：深度思考开关，展示 AI 推理过程（可折叠查看）
- **附件与多模态**：图片 / 文档上传（拖拽 / 粘贴 / 点击），支持视觉理解和文档文本提取
- **图像生成与编辑**：文生图、图像 Inpainting
- **Markdown 渲染**：完整 Markdown（表格 / 列表 / 链接 / Mermaid / KaTeX）
- **代码展示**：语法高亮、一键复制
- **消息反馈**：点赞 / 点踩 / 复制
- **会话共享**：生成唯一分享链接（UUID Token），只读浏览
- **技能快捷栏**：底部 SkillBar 展示常用技能，`@` 前缀触发联想补全
- **设置页面**：模型偏好、深色模式、发送快捷键、语言、字号、数据与隐私（导出对话 / 清除对话）
- **响应式布局**：PC 左右分栏 + 移动端适配（768px 断点）
- **后台继续生成**：用户切换页面后对话继续完成，回来时自动恢复
- **推荐问题（G4 基础）**：欢迎页展示引导性推荐问题

### 3.3 G3 — 能力扩展

通过可插拔的扩展机制，让 AI 从"纯对话"进化为"能做事"的智能助手。

**功能要点**：

- **技能系统（Skill）**：以 Markdown 编写可复用的 AI 行为指令，注入 System Prompt，支持 `@` 引用递归嵌套（最大深度 3 层）和触发词自动激活
- **函数调用（原生工具）**：`[ToolDescription]` 反射注册 .NET 方法为 AI 可调用工具，`ToolChatClient` 自动循环执行
- **工具审批机制**：通过 `IToolApprovalProvider` 接口在工具执行前拦截审批，可自行实现弹窗 / 日志 / 权限检查等审批逻辑
- **内置工具服务**（`NewLife.AI/Tools/`）：
  - `ISearchService` — Bing / DuckDuckGo / Serper / Remote 联网搜索
  - `ITranslateService` — MyMemory / Remote 文本翻译
  - `IWeatherService` — NMC（中国）/ Wttr（全球）/ Remote
  - `IWebFetchService` — 直接抓取 / Remote 代理
  - `IIpLocationService` — IpApi / Pconline / Remote
  - `NetworkToolService` — Ping / DNS 综合网络工具
  - `HolidayToolService` — 节假日 / 节气（位于 ChatData）
  - `CurrentUserTool` — 当前用户档案（位于 ChatData）
- **MCP 工具**：
  - 客户端对接外部 MCP 服务器（stdio / HTTP SSE）
  - 服务端通过 `AspNetMcpServer` 将本系统工具暴露为 MCP 服务
  - 多 Server 工具名冲突时自动加前缀去重
- **过滤器链（IChatFilter）**：洋葱圈模型，在对话前后插入自定义逻辑；开源版内置 `LearningFilter`（自动注入记忆、触发自学习分析）
- **规划器（Planner）**：`FunctionCallingPlanner` 将目标分解为多步工具调用并执行

### 3.4 G4 — 知识进化（基础版）

让 AI 从"对话工具"初步进化为"越用越懂你"的助手，自动从对话中提取结构化用户记忆。

**功能要点（开源基础版）**：

- **用户记忆（10 分类）**：自动从对话中提取结构化记忆并持久化，对标 Mem0 / ChatGPT Memory / Claude Projects：
  1. 身份信息 · 2. 偏好 · 3. 习惯 · 4. 兴趣 · 5. 背景
  6. 职业 · 7. 目标 · 8. 人际关系 · 9. 技能专长 · 10. 交互指令
- **对话分析引擎**：`ConversationAnalysisService` 异步分析对话内容，结构化提取知识，评估置信度
- **记忆注入**：`LearningFilter.before` 每次对话前自动将相关记忆注入 System Prompt
- **记忆管理 API**：列表查询、手动添加 / 编辑 / 删除、分类筛选
- **推荐问题**：欢迎页引导性推荐问题，由管理员在后台配置

> **以下高级能力在当前版本中暂不包含**：知识蒸馏（Dream）、痛觉记忆（Nociception）、好奇心探索（Curiosity）、知识图谱（MemoryEntity + MemoryRelation）、每日简报（DailyBriefing）、知识融合、信任分级与知识审核、知识库 RAG、记忆检索增强（时间衰减 + MMR）。

### 3.5 G5 — API 网关

API 网关面向外部系统提供标准化的 AI 对话接口，核心价值是在接口转发之外提供**记忆注入与技能增强**。

**功能要点**：

- **多协议入站**：
  - OpenAI `/v1/chat/completions` — 聊天补全，支持流式 / 函数调用 / 视觉
  - OpenAI Responses `/v1/responses` — 推理模型（o3 / gpt-5 等）
  - Anthropic `/v1/messages` — Claude 系列
  - Google Gemini `/v1/gemini` — Gemini 系列
  - 图像生成 `/v1/images/generations`
  - 图像编辑 `/v1/images/edits`（multipart/form-data）
  - 模型发现 `/v1/models`
- **全功能对等**：打开所有开关的情况下，数据流处理与 WebUI 完全一致（走 `ChatPipeline`，记忆注入、技能增强、工具调用、过滤器链）
- **AppKey 认证**：`sk-` 前缀随机密钥，创建时仅展示一次；每个 AppKey 绑定用户身份
- **智能路由**：根据模型名自动匹配提供商与模型
- **上游 429 指数退避重试**：失败时带随机抖动，最多 5 次
- **用量记录**：每次调用记录 Token 消耗、耗时等指标，同时归属到用户与 AppKey 双维度

### 3.6 G6 — 桌面客户端（不含）

> **本目标在当前版本中暂不提供实现**。可通过 `IToolApprovalProvider` 接口 + `ChatClientBuilder.UseTools(approvalProvider, providers)` 自行实现桌面宿主与工具审批 UI。

### 3.7 G7 — 办公平台集成

将 AI 对话能力延伸至企业即时通信平台。

**功能要点**：

- `IMessageChannel` 统一抽象（`NewLife.AI/Channels/`）
- **四个内置渠道实现**：
  - `DingTalkChannel` — 钉钉机器人，支持 Markdown 格式
  - `WeComChannel` — 企业微信应用消息回调
  - `FeishuChannel` — 飞书消息事件 + 卡片消息
  - `WebhookChannel` — 通用 HTTP 回调
- 每个渠道实现独立的签名 / Token 验证
- **回调入口**：`POST /api/channel/{code}/callback`

> 渠道配置管理页暂不包含于本版本。

### 3.8 G8 — 智能体框架

提供规划器、多智能体编排等基础设施，支持复杂任务协作。

**功能要点**：

- **`IAgent` 统一抽象**
- **`ConversableAgent`**：可与工具交互的单智能体
- **`GroupChat`**：多智能体群聊，支持 RoundRobin 轮询选择器 / LLM Selector
- **`ParallelGroupChat`**：多智能体并行执行
- **`DelegatingAgent`**：智能体委派装饰器
- **`AgentAsTool`**：将智能体封装为可被其他智能体调用的工具
- **`FunctionCallingPlanner`**：将自然语言目标分解为多步工具调用计划并执行
- **`AgentMessage`**：多智能体协作消息模型（文本 / 工具调用 / 结果）

## 4. 非功能需求

### 4.1 性能

| 指标 | 目标 |
|------|------|
| 首 Token 响应（流式） | ≤ 2 秒（受上游模型影响）|
| 会话列表加载 | ≤ 500 ms（20 条）|
| API 网关转发额外开销 | ≤ 50 ms |
| SSE 断线重连 | 指数退避 3 次，恢复时间 ≤ 10 秒 |
| 单实例并发对话 | ≥ 100（受模型厂商配额限制）|

### 4.2 安全

- **AppKey**：`sk-` 前缀随机 40 字符，创建时仅展示一次（不可重复查看）
- **用户认证**：由 `NewLife.Cube` 权限框架处理
- **敏感信息保护**：服务商 API Key 在数据库加密存储
- **工具审批接口**：`IToolApprovalProvider` 可拦截工具执行前确认
- **分享链接**：UUID Token 不可猜测，有效期可配置（默认 30 天，0 为永久）

### 4.3 兼容性

- **NewLife.AI** / **NewLife.ChatData**：netstandard2.1，实际可在 .NET 4.5 至 .NET 10 上运行
- **NewLife.ChatAI**：net8.0 / net10.0（ASP.NET Core）
- **NewLife.AI.Extensions**：net6.0 / net8.0 / net10.0
- **浏览器**：Chrome / Edge / Safari / Firefox 最新两个大版本
- **数据库**：SQLite（默认）/ MySQL / SQL Server / PostgreSQL / Oracle（由 XCode 统一适配）

### 4.4 可扩展性

- 新增 AI 服务商：继承 `AiClientBase` 或 `OpenAIChatClient`，添加 `[AiClient]` 特性，零配置自动注册
- 新增工具：在 .NET 方法上添加 `[ToolDescription]` 特性，调用 `ToolRegistry.AddTools<T>()` 即可
- 新增过滤器：实现 `IChatFilter`，通过 `ChatClientBuilder.UseFilters()` 挂入管道
- 新增上下文增强器：实现 `IContextEnricher`，通过 DI 注册即被 `MessageFlow.Prepare` 按 `Order` 升序调用
- 新增协议：继承 `AiClientBase`，重写请求/响应转换方法

## 5. 边界与约束

### 5.1 开源版包含的能力

| 分类 | 能力 |
|------|------|
| SDK 基础 | 46 服务商 / 5 协议 / IChatClient / ChatClientBuilder / Filters / Tools / MCP / Agents / Planner / Memory / VectorStore |
| 数据层 | 用户记忆（10 类） / 技能 / 推荐问题 / ChatPipeline / MessageFlow / IContextEnricher 接口 |
| Web 应用 | 20 API 控制器 / 网关多协议 / AppKey / 用量统计 / 内嵌 React 前端 / 魔方管理后台 |
| 扩展点 | 所有接口与抽象类开放，方便二次开发 |

### 5.2 当前版本**暂不包含**的能力

- **桌面宿主客户端**：本版本不含，`IToolApprovalProvider` 接口已预留，可自行实现本机工具与审批 UI
- **L4 知识进化高级服务**：
  - Dream 离线蒸馏（`MemoryDistillationService`）
  - 痛觉记忆（`NociceptionService` + `NociceptionMemory` 实体）
  - 好奇心探索（`CuriosityService` + `CuriosityQueue` 实体）
  - 知识图谱（`MemoryGraphService` + `MemoryEntity` + `MemoryRelation` 实体）
  - 每日简报（`DailyBriefingService`）
  - 知识融合（`KnowledgeFusionService`，碎片进化为技能）
  - 信任分级与知识审核（`TrustService` + `KnowledgeReview` / `KnowledgeHistory` 实体）
  - 智能体自我进化（`LessonsLearned`）
- **知识库 RAG**：`KnowledgeBase` / `KnowledgeDocument` / `KnowledgeArticle` 实体 + 多格式文档解析 + LLM 清洗 + `KnowledgeFilter` 自动注入
- **管道增强**（暂不含）：
  - `ChatPipeline` 继承实现：轻量模型路由、SSE 事件中转、OpenTelemetry 埋点、后台继续生成衔接
  - `ToolDiscoveryProvider`：渐进式工具发现（Advertise-Load-Read），工具数超阈值时自动降级为摘要目录
  - 三个内置 `IContextEnricher`：`EarlyTruncationEnricher`（Order=50）/ `SandwichEnricher`（Order=100）/ `CompactionEnricher`（Order=200）
- **魔方后台管理页**：Evolution 九个服务管理页、Knowledge 七个服务管理页、多智能体管理页、定时对话任务（ChatJob）管理页、渠道数据库管理页
- **AgentTriggerFilter**：智能体触发过滤器
- **IContextEnricher 内置实现**：仅提供接口与 `MessageFlow` 装配钩子，内置三件套暂不包含

### 5.3 技术约束

- 本系统不负责用户登录鉴权，必须集成 `NewLife.Cube` 或自实现权限框架
- 数据库必须支持 XCode ORM（SQLite / MySQL / SQL Server / PostgreSQL / Oracle 等）
- 前端静态资源通过 `ChatAiStaticFilesService` 内嵌到 DLL 托管，不依赖外部 CDN
- MCP Server 若使用 stdio 传输，宿主必须有权启动子进程
- 高级进化能力（见 §5.2）参考架构扩展点说明，可通过实现相应接口自行扩展

## 6. 术语表

| 术语 | 含义 |
|------|------|
| 会话（Conversation） | 一次完整的多轮对话上下文 |
| 消息（Message） | 会话中的单条发言，分用户消息和 AI 回复 |
| 模型（Model） | 后端接入的大语言模型，如 GPT-4o、Qwen-Max 等 |
| 流式输出（Streaming） | 服务端逐 token 推送响应内容（SSE） |
| 思考模式（Thinking Mode） | 控制模型推理深度：Auto / Think / Fast |
| API 网关（API Gateway） | 对外提供统一的 AI 模型访问接口，兼容多种标准协议 |
| 函数调用（Function Calling） | 模型调用外部工具 / 函数的能力，Agent 的基础 |
| MCP（Model Context Protocol） | 模型上下文协议，AI 应用与工具 / 数据源交互的标准协议 |
| 交错思考（Interleaved Thinking） | 工具调用期间回传思考过程，形成 think–tool–think 链路 |
| 多模态（Multi-modal） | 支持文本、图像、视觉理解、图像生成等多种媒体形式 |
| AppKey | 外部系统通过 API 网关调用模型服务时使用的访问凭证 |
| 技能（Skill） | 可复用的 AI 行为指令，以 Markdown 格式编写，注入 System Prompt |
| 记忆（Memory） | 从对话中提取并持久化的知识片段，包括用户偏好、业务规则、领域知识等 |
| 对话分析（Conversation Analysis） | 后台对对话内容进行结构化知识提取与分类 |
| MessageFlow | `ChatData/Services/MessageFlow.cs` 定义的消息处理五阶段模板：Validate→Prepare→Execute→Persist→PostProcess |
| IContextEnricher | 在 Prepare 阶段对上下文消息列表进行修饰的可插拔扩展点（接口） |
| IChatPipeline | 管道抽象接口（位于 NewLife.AI）；`ChatPipeline`（位于 NewLife.ChatData）为基础实现 |
| IChatFilter | 洋葱圈过滤器接口，挂入 `ChatClientBuilder.UseFilters()` |
| IToolProvider | 工具提供者统一接口，由 `ToolRegistry` / `McpClientService` / `DbToolProvider` 实现 |
| IToolApprovalProvider | 工具执行前审批拦截接口 |

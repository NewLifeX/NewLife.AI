# 对话交互式 AI 系统 — 需求规格说明

> 版本：v0.1（草稿）  
> 日期：2026-02-28  
> UI 参考：<https://chat.qwen.ai>

---

## 1. 概述

### 1.1 目标

构建一个面向终端用户的多模型对话交互式 AI 系统，提供自然语言问答、多轮对话、附件上传、模型切换、消息流式输出等核心能力。系统以 Web 前端为主要交互界面，后端对接多种大语言模型（LLM）。

### 1.2 用户角色

| 角色 | 说明 |
|------|------|
| 普通用户 | 注册/登录后使用对话功能 |
| 访客 | 通过分享链接查看共享对话（只读） |
| 管理员 | 管理用户、模型配置、系统设置（后续迭代） |

### 1.3 术语

| 术语 | 含义 |
|------|------|
| 会话（Conversation） | 一次完整的多轮对话上下文 |
| 消息（Message） | 会话中的单条发言，分用户消息和系统消息 |
| 模型（Model） | 后端接入的大语言模型，如 GPT-4o、Qwen-Max 等 |
| 流式输出（Streaming） | 服务端逐 token / 逐行推送响应内容 |
| 思考模式（Thinking Mode） | 控制模型推理深度的策略：自动 / 思考 / 快速 |

## 3. 侧边栏（左侧面板）

### 3.1 新建对话

- 位于侧边栏 **顶部**。
- 点击后在主内容区创建一个新的空白会话，清空对话内容区，聚焦到输入框。
- 当前会话若有内容，自动保存到对话列表。

### 3.2 对话列表

- 位于新建对话按钮下方，占据侧边栏主体区域。
- **排序**：按最后活跃时间 **从近到远** 排列。
- **分组标签**：今天、昨天、过去 7 天、过去 30 天、更早。
- **默认显示**：初始加载最近 20 条会话摘要（显示标题 + 最后消息时间）。
- **滚动加载**：用户向下滚动时自动加载更多（每次追加 20 条），直到无更多数据。
- **当前会话高亮**：当前打开的会话项带有高亮 / 选中态样式。
- **会话操作**：鼠标悬停时显示操作菜单（三点图标），支持：
  - 重命名会话
  - 删除会话（二次确认）
  - 置顶 / 取消置顶

### 3.3 个人图标

> 参考截图：
> ![个人图标与菜单](qwen2.png)

- 位于侧边栏 **底部**，显示用户头像或首字母圆形图标。
- **点击展开菜单**，菜单项自上而下依次为：
  1. **账号名**：显示当前登录用户名称（不可点击，仅展示）。
  2. **设置**：点击后弹出设置页（模态弹窗或全屏覆盖）。

## 5. 主内容区（右侧）

### 5.1 模型选择器

> 参考截图：
> ![模型选择器下拉](qwen4.png)

- 位于主内容区 **左上角**。
- 以下拉菜单形式展示，显示当前选中模型名称。
- 点击展开模型列表：
  - 每个模型项显示 **模型名称**（如 Qwen-Max、GPT-4o、DeepSeek-R1 等）。
  - 选中后立即切换，后续对话使用新模型。
  - 已发送的消息不受模型切换影响。
- **仅支持选择模型**，不支持对比、更多等附加功能。

### 5.2 新会话页（空状态）

当无对话内容时，主内容区显示新会话欢迎页：

- 中央显示欢迎语 / 品牌标识。
- 可选：展示推荐问题卡片（如"帮我写一封邮件"、"解释量子计算"等），点击即发送。
- 底部为输入区。

### 5.3 输入区

> 参考截图：
> ![新会话页与输入区](qwen5.png)

输入区固定在主内容区底部，包含以下元素（从左到右）：

```
┌─────────────────────────────────────────────────────┐
│ [+]  │      输入框（多行，自动伸缩）       │ [模式▼] [发送] │
└─────────────────────────────────────────────────────┘
```

#### 5.3.1 附件上传（+ 号按钮）

- 点击弹出文件选择器，支持上传附件。
- 支持的文件类型：图片（jpg/png/gif/webp）、文档（pdf/docx/txt/md/csv）。
- 单文件大小限制：20 MB。
- 单次最多上传 5 个文件。
- 上传后在输入框上方以缩略图 / 文件名标签展示，支持移除。
- 其他高级功能（语音输入、网页搜索等）暂不支持，将来完善。

#### 5.3.2 输入框

- 支持多行文本输入，高度随内容自动伸缩（最小 1 行，最大 8 行后出现滚动条）。
- 支持粘贴图片（自动作为附件处理）。
- 支持 Markdown 语法输入。
- 空内容时发送按钮置灰禁用。

#### 5.3.3 思考模式选择器

> 参考截图：
> ![思考模式选择器](qwen6.png)

- 位于输入框右侧，以下拉按钮形式展示。
- 选项：
  - **自动**（默认）：由系统根据问题复杂度自动决定推理深度。
  - **思考**：启用深度推理（Chain-of-Thought），适合复杂推理 / 数学 / 编程题。
  - **快速**：跳过深度推理，快速返回结果，适合简单问答。
- 选择后当前会话内保持，切换会话时恢复为默认。

#### 5.3.4 发送按钮

- 输入框有内容时激活，点击或按快捷键发送消息。
- 发送后清空输入框，回答生成期间按钮变为 **停止** 按钮（■），可中断生成。

## 6. 对话内容区

### 6.1 消息布局

- 用户消息：右对齐，带用户头像，背景色区分。
- 系统消息（AI 回复）：左对齐，带模型图标，背景色区分。
- 每条消息显示发送时间（悬停时显示精确时间）。

### 6.2 流式输出

- 用户提问后，系统消息区域立即出现，内容 **逐行 / 逐段实时输出**（打字机效果）。
- 输出过程中消息区域自动滚动到底部，用户手动上滚时暂停自动滚动。
- 输出完成前显示加载指示器（闪烁光标或脉冲点）。

### 6.3 Markdown 渲染

消息内容以 Markdown 格式渲染，特别支持：

| 元素 | 说明 |
|------|------|
| 标题 | h1–h6 |
| 列表 | 有序 / 无序 / 嵌套列表 |
| 表格 | 标准 Markdown 表格，支持对齐 |
| 代码片段 | 行内代码 + 代码块（语法高亮 + 一键复制按钮）|
| 流程图 | 支持 Mermaid 语法渲染为交互式流程图 |
| 数学公式 | 支持 KaTeX / LaTeX 渲染 |
| 链接 / 图片 | 可点击跳转 / 内联预览 |
| 引用块 | 块引用样式 |

### 6.4 消息操作

> 参考截图：
> ![消息操作按钮（复制与编辑）](qwen7.png)

#### 6.4.1 通用操作（用户消息 & 系统消息均支持）

- **复制**：复制消息原始文本（Markdown 源码）到剪贴板。
- **编辑**：点击后消息变为可编辑状态；
  - 用户消息编辑后重新发送，触发新的 AI 回复（原回复保留为历史分支）。
  - 系统消息编辑后仅修改显示内容，不触发新请求。
  - 编辑时显示"保存"和"取消"按钮。

操作入口：鼠标悬停消息时，在消息右上角 / 底部浮现操作图标。

#### 6.4.2 系统消息专属操作

系统消息（AI 回复）下方显示一排操作按钮，从左到右依次为：

| 按钮 | 图标 | 说明 |
|------|------|------|
| 复制 | 📋 | 复制回复全文到剪贴板 |
| 点赞 | 👍 | 对回复表示满意；点击后高亮，再次点击取消 |
| 点踩 | 👎 | 对回复表示不满意；可弹出反馈输入框收集原因 |
| 分享 | 🔗 | 创建共享链接（详见 6.5 节） |
| 重新生成 | 🔄 | 使用当前模型重新生成回复；新回复替换当前回复，旧回复可通过左右箭头切换 |

### 6.5 点赞 / 点踩数据

- 点赞与点踩数据将上报服务端，**可用于模型学习训练**。
- 用户可在设置页的"数据与隐私"中关闭此功能（关闭后点赞 / 点踩仍可操作，但数据不上报）。
- 点踩时可选填反馈原因（多选标签 + 自由文本）：
  - 回答不准确
  - 回答不完整
  - 格式混乱
  - 内容有害
  - 其他（自由输入）

### 6.6 分享功能

> 参考截图：
> ![分享对话弹窗](qwen8.png)

- 点击"分享"按钮后：
  1. 系统生成一个唯一的 **共享链接**（URL）。
  2. 弹窗展示链接，用户可一键复制。
  3. **共享范围**：创建链接后新发送的消息 **不会被共享**，仅包含创建时刻之前的对话内容。
  4. 拥有 URL 的任何用户均可查看共享对话（**只读**，无需登录）。
- 共享对话页面：
  - 顶部显示"共享对话"标识 + 分享者信息。
  - 完整展示共享范围内的对话内容（Markdown 渲染）。
  - 不显示操作按钮（点赞 / 编辑等）。
  - 底部显示"使用 XXX 开始你的对话"引导。

## 7. 数据模型（概要）

### 7.1 核心实体

```
User
├── Id : Long
├── Name : String
├── Avatar : String
├── Settings : JSON
└── CreateTime : DateTime

Conversation
├── Id : Long
├── UserId : Long
├── Title : String
├── ModelCode : String          // 当前使用的模型
├── IsPinned : Boolean          // 是否置顶
├── LastMessageTime : DateTime
├── CreateTime : DateTime
└── UpdateTime : DateTime

Message
├── Id : Long
├── ConversationId : Long
├── Role : Enum(User, Assistant)
├── Content : String            // Markdown 文本
├── ThinkingMode : Enum(Auto, Think, Fast)
├── ParentMessageId : Long?     // 编辑 / 重新生成时的分支引用
├── Attachments : JSON          // 附件列表
├── CreateTime : DateTime
└── UpdateTime : DateTime

MessageFeedback
├── Id : Long
├── MessageId : Long
├── UserId : Long
├── Type : Enum(Like, Dislike)
├── Reason : String?            // 点踩原因
├── AllowTraining : Boolean     // 是否允许用于训练
└── CreateTime : DateTime

SharedConversation
├── Id : Long
├── ConversationId : Long
├── ShareToken : String         // 唯一分享标识（生成 URL 用）
├── SnapshotMessageId : Long    // 快照截止到的最后一条消息 ID
├── CreatorUserId : Long
├── CreateTime : DateTime
└── ExpireTime : DateTime?      // 过期时间，null 表示永不过期
```

### 7.2 模型配置

```
ModelConfig
├── Id : Long
├── Code : String               // 模型编码（唯一标识）
├── Name : String               // 显示名称
├── Provider : String           // 提供商（OpenAI / Alibaba / DeepSeek 等）
├── Endpoint : String           // API 地址
├── MaxTokens : Int32           // 最大 Token 数
├── SupportThinking : Boolean   // 是否支持思考模式
├── SupportVision : Boolean     // 是否支持图片输入
├── IsEnabled : Boolean         // 是否启用
├── Sort : Int32                // 排序序号
└── CreateTime : DateTime
```

## 9. 非功能性需求

### 9.1 性能

- 首条 token 响应延迟（TTFT）：≤ 2 秒（不含模型推理）。
- 流式输出帧率：≥ 10 token/秒 的渲染刷新。
- 会话列表首屏加载：≤ 500 ms。
- 附件上传：支持断点续传，进度条可视。

### 9.2 可用性

- 系统可用性目标：99.9%。
- 模型服务降级策略：主模型不可用时自动切换到备选模型，并通知用户。

### 9.3 安全

- 全站 HTTPS。
- 本期不启用登录鉴权；接口预留 Bearer Token / OAuth 2.0 接入扩展点。
- XSS 防护：Markdown 渲染时过滤危险标签和脚本。
- 速率限制：按用户维度限流，防滥用。
- 共享链接：Token 使用不可猜测的随机值（UUID / NanoID）。

### 9.4 兼容性

- 浏览器：Chrome / Edge / Firefox / Safari 最新两个大版本。
- 移动端：iOS Safari / Android Chrome 响应式适配。

## 10. 工程实施约束（补充）

### 10.1 项目与依赖

- 新增独立 Web 项目：`NewLife.ChatAI`。
- 项目类型：ASP.NET Core Web（建议与现有仓库保持一致，目标框架 `net8.0`）。
- 必须引用：`NewLife.Cube.Core`。
- 前后端代码均放在 `NewLife.ChatAI` 项目内，采用同一站点托管（前端静态资源 + API）。

### 10.2 前端落地范围

- 前端 UI 风格与核心交互参考 `<https://chat.qwen.ai/>`。
- 必须覆盖本说明第 2～6 章全部能力（侧边栏、设置、模型选择、输入区、流式输出、消息操作、分享等）。
- 本期只实现必要功能，不引入未在本文定义的扩展能力（如对比模式、管理后台入口等）。

### 10.3 后端接口落地要求

- 在 `NewLife.ChatAI/Controllers` 目录创建控制器并提供前端所需的全部接口。
- 接口覆盖范围以第 8 章为准，不得缺项；可按职责拆分控制器：
  - `ConversationsController`
  - `MessagesController`
  - `FeedbackController`
  - `ShareController`
  - `AttachmentsController`
  - `ModelsController`
  - `UserSettingsController`
- 发送消息接口需支持流式返回（SSE 或等价流式机制），满足第 6.2 节实时输出要求。

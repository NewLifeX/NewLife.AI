# 模型供应商多模态能力矩阵

> 基于当前代码实现与已核对的官方资料整理。原则：**实现对应能力接口 = 支持；未实现 = 不支持（或暂未接入）**。
>
> 统计范围：`NewLife.AI/Clients` 下主客户端实现；测试范围对应 `XUnitTest/Clients`、`XUnitTest/Gateway`。

## 1. 统一抽象约定

- `IChatClient` 只保留对话能力（非流式/流式）
- 多模态通过能力接口按需实现：
  - `IImageClient`：文生图、图像编辑
  - `IVideoClient`：视频任务提交、任务状态查询
  - `ISpeechClient`：TTS 语音合成
  - `ITranscriptionClient`：ASR/STT 语音识别
  - `IRerankClient`：重排序
  - `IModelListClient`：模型列表

## 5. 本轮更新（2026-04-19）

- `GeminiChatClient` 已实现 `IImageClient`（Imagen4 文生图）和 `IModelListClient`（`GET /v1beta/models`）。
- `GeminiChatClient` 移除 `imagen-3.0-generate-001` 模型声明，统一以 Imagen4 为主。
- `AzureAIChatClient` 显式覆盖视频接口并抛出 `NotSupportedException`，与 `IVideoClient` 约定保持一致。
- `DeepSeekChatClient` 已显式禁用文生图/图像编辑/TTS/STT/视频能力，仅保留对话与模型列表。

## 6. 当前落地策略

1. 不扩展 `IChatClient` 主接口，保持兼容稳定。
2. 按能力接口进行实现与判定：
   - 支持能力：实现对应接口并补测试。
   - 不支持能力：不实现接口，并在注释/文档中明确标注。
3. 集成测试按能力分组推进：文本 → 工具调用 → 视觉/音频 → 图像/视频长任务。

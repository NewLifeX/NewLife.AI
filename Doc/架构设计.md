## 2. 项目结构与依赖

### 2.1 项目信息

- 项目名称：`NewLife.ChatAI`
- 项目类型：ASP.NET Core Web
- 目标框架：`net8.0`
- 必须引用：`NewLife.Cube.Core`
- 前后端同项目：前端静态资源放 `wwwroot/`，API 放 `Controllers/`

| 字段 | 类型 | 说明 |

| Id | Int64 | 主键，自增 |
| UserId | Int32 | 用户编号（来自外部登录模块） |
| Title | String(200) | 标题，显示在侧边栏 |

| ThinkingMode | Int32 | 思考模式：Auto=0, Think=1, Fast=2 |

#### 对话消息（ChatMessage）

| 字段 | 类型 | 说明 |

#### 消息反馈（MessageFeedback）

| AllowTraining | Boolean | 是否允许用于训练 |

#### 共享会话（SharedConversation）

| 字段 | 类型 | 说明 |

| CreatorUserId | Int32 | 分享发起用户 |

索引：`ShareToken`（唯一）、`ConversationId`、`CreatorUserId`

#### 模型配置（ModelConfig）

| 字段 | 类型 | 说明 |

| Id | Int32 | 主键，自增 |

| AllowTraining | Boolean | 是否允许反馈数据用于训练 |

索引：`UserId`（唯一）

### 3.2 附件存储

附件 **不** 使用独立的 `ChatAttachment` 表，而是复用 **魔方（NewLife.Cube）已有的 `Attachment` 表**。

魔方附件表关键字段：

| 字段 | 说明 |
|------|------|
| Id | Int64，雪花 ID |
| Category | 业务分类，填 `ChatAI` |
| Key | 业务主键，可存消息 ID |
| FileName | 原始文件名 |
| Extension | 扩展名 |
| Size | 文件大小 |
| ContentType | MIME 类型 |
| FilePath | 本地相对路径 |

使用方式：
- 上传附件时，`Category` 设为 `"ChatAI"`，`Key` 设为消息 ID（发送前为 `"temp_用户ID"`）。
- 消息的 `Attachments` 字段存储附件 ID 数组的 JSON，如 `[123456, 789012]`。
- 下载/预览通过魔方附件控制器，或封装为 `/api/attachments/{id}` 代理。

所有接口错误均返回以下 JSON 结构：

```json
{
  "code": "ERROR_CODE",
  "message": "人类可读的错误描述",
  "traceId": "链路追踪ID"
}
```

常用错误码：

| 错误码 | HTTP 状态码 | 说明 |
|--------|-----------|------|
| `CONVERSATION_NOT_FOUND` | 404 | 会话不存在 |
| `MESSAGE_NOT_FOUND` | 404 | 消息不存在 |
| `MODEL_UNAVAILABLE` | 503 | 模型服务不可用 |

## 5. SSE 流式协议规范

### 5.1 协议概述

发送消息接口 `POST /api/conversations/{id}/messages` 以 **Server-Sent Events（SSE）** 格式返回响应。

HTTP 响应头：

```
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
```

每条 SSE 事件格式：

```
data: {JSON}\n\n
```

### 5.2 请求格式

```json
{
  "content": "用户输入的消息文本",
  "thinkingMode": 0,
  "attachmentIds": [123456, 789012]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| content | String | 是 | 消息文本内容 |
| thinkingMode | Int32 | 否 | 0=自动(默认), 1=思考, 2=快速 |
| attachmentIds | Int64[] | 否 | 附件 ID 列表 |

### 5.3 SSE 事件类型

#### 5.3.1 消息头事件（message_start）

流开始时发送，包含消息的元信息：

```
data: {"type":"message_start","messageId":5001,"model":"qwen-max","thinkingMode":1}
```

| 字段 | 说明 |
|------|------|
| type | 固定 `"message_start"` |
| messageId | AI 回复的消息 ID |
| model | 实际使用的模型编码 |
| thinkingMode | 实际使用的思考模式 |

#### 5.3.2 思考过程增量事件（thinking_delta）

思考模式下，输出模型的推理过程：

```
data: {"type":"thinking_delta","content":"让我分析一下这个问题..."}
```

| 字段 | 说明 |
|------|------|
| type | 固定 `"thinking_delta"` |
| content | 思考内容的增量文本 |

#### 5.3.3 思考完成事件（thinking_done）

思考过程结束：

```
data: {"type":"thinking_done","thinkingTime":3200}
```

| 字段 | 说明 |
|------|------|
| type | 固定 `"thinking_done"` |
| thinkingTime | 思考耗时，毫秒 |

#### 5.3.4 内容增量事件（content_delta）

正式回答的增量内容：

```
data: {"type":"content_delta","content":"这是回答的一部分"}
```

| 字段 | 说明 |
|------|------|
| type | 固定 `"content_delta"` |
| content | 回答内容的增量文本 |

#### 5.3.5 完成事件（message_done）

全部内容输出完毕：

```
data: {"type":"message_done","usage":{"promptTokens":150,"completionTokens":320,"totalTokens":470}}
```

| 字段 | 说明 |
|------|------|
| type | 固定 `"message_done"` |
| usage | Token 用量统计（可选） |
| usage.promptTokens | 提示 Token 数 |
| usage.completionTokens | 回复 Token 数 |
| usage.totalTokens | 总 Token 数 |

#### 5.3.6 错误事件（error）

流式输出过程中发生错误：

```
data: {"type":"error","code":"MODEL_UNAVAILABLE","message":"模型服务暂时不可用，请稍后重试"}
```

| 字段 | 说明 |
|------|------|
| type | 固定 `"error"` |
| code | 错误码 |
| message | 错误描述 |

### 5.4 完整 SSE 流示例

#### 快速模式（无思考过程）

```
data: {"type":"message_start","messageId":5001,"model":"qwen-max","thinkingMode":2}

data: {"type":"content_delta","content":"你好！"}

data: {"type":"content_delta","content":"我是AI助手，"}

data: {"type":"content_delta","content":"有什么可以帮你的？"}

data: {"type":"message_done","usage":{"promptTokens":10,"completionTokens":15,"totalTokens":25}}
```

#### 思考模式（含思考过程）

```
data: {"type":"message_start","messageId":5002,"model":"deepseek-r1","thinkingMode":1}

data: {"type":"thinking_delta","content":"用户问的是一个数学问题，"}

data: {"type":"thinking_delta","content":"我需要用二次方程求根公式..."}

data: {"type":"thinking_delta","content":"判别式 Δ = b²-4ac = ..."}

data: {"type":"thinking_done","thinkingTime":3200}

data: {"type":"content_delta","content":"根据求根公式，方程的解为：\n\n"}

data: {"type":"content_delta","content":"$$x = \\frac{-b \\pm \\sqrt{b^2-4ac}}{2a}$$"}

data: {"type":"message_done","usage":{"promptTokens":50,"completionTokens":120,"totalTokens":170}}
```

#### 错误中断

```
data: {"type":"message_start","messageId":5003,"model":"gpt-4o","thinkingMode":0}

data: {"type":"content_delta","content":"让我来回答"}

data: {"type":"error","code":"MODEL_UNAVAILABLE","message":"模型连接超时，请稍后重试"}
```

### 5.5 前端处理逻辑

1. 收到 `message_start`：创建消息气泡，显示模型图标和加载指示器。
2. 收到 `thinking_delta`：在折叠区域追加思考过程文本，显示"思考中..."标题。
3. 收到 `thinking_done`：关闭思考区域，标题改为"思考过程（耗时 X.X 秒）"。
4. 收到 `content_delta`：在正式回答区域追加内容，进行 Markdown 增量渲染。
5. 收到 `message_done`：移除加载指示器，显示操作按钮，可展示 Token 用量。
6. 收到 `error`：停止流，显示错误提示，已输出内容保留。
7. 连接断开且未收到 `message_done` / `error`：视为异常中断，保留已有内容并提示。

## 6. 系统配置

### 6.1 ChatSetting 配置类

继承 `Setting<ChatSetting>`，提供系统级配置。配置文件默认位于 `Config/ChatSetting.config`。

```csharp
public class ChatSetting : Setting<ChatSetting>
{
    /// <summary>分享有效期。共享链接有效天数，0表示永不过期，默认30天</summary>
    public Int32 ShareExpireDays { get; set; } = 30;

    /// <summary>默认模型。新用户的默认模型编码</summary>
    public String DefaultModel { get; set; } = "qwen-max";

    /// <summary>默认思考模式。Auto=0, Think=1, Fast=2</summary>
    public Int32 DefaultThinkingMode { get; set; } = 0;

    /// <summary>上下文轮数。默认10轮</summary>
    public Int32 DefaultContextRounds { get; set; } = 10;

    /// <summary>最大附件大小。单位MB，默认20</summary>
    public Int32 MaxAttachmentSize { get; set; } = 20;

    /// <summary>最大附件数。单次最多上传附件数，默认5</summary>
    public Int32 MaxAttachmentCount { get; set; } = 5;

    /// <summary>标题自动生成。首条消息后是否自动生成会话标题</summary>
    public Boolean AutoGenerateTitle { get; set; } = true;

    /// <summary>标题生成提示词。用于生成标题的提示模板</summary>
    public String TitlePrompt { get; set; } = "请用10个字以内总结以下对话的主题，只输出标题文字，不要标点符号：";

    /// <summary>允许的文件扩展名。逗号隔开</summary>
    public String AllowedExtensions { get; set; } = ".jpg,.jpeg,.png,.gif,.webp,.pdf,.docx,.txt,.md,.csv";
}
```

### 6.2 配置使用说明

- 修改配置后自动保存到 `Config/ChatSetting.config`。
- 系统启动时自动加载，不存在则创建默认配置文件。

## 8. 标题自动生成策略

### 8.1 触发条件

1. 会话的 `MessageCount == 0`（首次发送消息时）。
2. 系统配置 `ChatSetting.AutoGenerateTitle == true`。
3. 会话标题为默认值"新建对话"。

### 8.2 实现流程

```
用户发送首条消息
    ├── 主流程：正常调用模型生成 AI 回复（SSE 流式）
    └── 异步任务：调用模型生成标题
         ├── 输入：用户首条消息内容
         ├── 提示词：ChatSetting.TitlePrompt + 消息内容
         ├── 使用模型：当前会话的模型（非流式调用）
         ├── 成功：更新会话标题
         └── 失败：保持默认标题，不影响主流程
```

### 8.3 前端交互

- 首条消息发送后，通过 SSE 事件或轮询获取会话标题更新。
- 建议在 `message_done` 事件中附加 `title` 字段（仅首条消息时返回）：

```
data: {"type":"message_done","usage":{...},"title":"关于量子计算的讨论"}
```

## 9. 设计决策记录

| # | 决策 | 理由 |
|---|------|------|
| 1 | 不支持消息分支 | 简化前后端复杂度，编辑/重新生成直接替换 |
| 2 | 不做模型降级 | 模型不可用时直接报错，避免用户不知情使用非预期模型 |
| 3 | 复用魔方附件表 | 避免重复建设附件管理，魔方附件表已有完善的存储和管理能力 |
| 4 | 登录鉴权由外部模块提供 | 职责分离，NewLife.Cube 已有完整的用户和权限体系 |
| 5 | 共享有效期由后台配置 | 统一管理，避免每个用户自行设置导致管理困难 |
| 6 | SSE 区分思考/回答事件 | 前端可独立渲染思考过程折叠区域和正式回答 |
| 7 | 标题异步生成 | 不阻塞消息响应流程，用户体验更好 |
